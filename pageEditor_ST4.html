Sitemap

Technical Description

This is a web page that is built in HTML and CSS and the result is a visual sitemap of the site. Semantic markup of the page include <nav> element to describe navigation, <footer> element to describe footer content and linking to other pages is done using <a> element. The primary sitemap chart is also presented in the form of SVG lines that interrelate various nodes representing the pages and depicting the hierarchy of pages.


Its design is based on Flexbox positioning. The .wrapper and .container classes are implemented with Flexbox to centerize content and form three content columns to compose the pages. The style of each page node is given using the .node class which is an orange background (rgba(255, 165, 0, 0.85)) and contains pad, rounded corner (border-radius) and shadow (box-shadow) as well as has scaled hover effect (transform: scale(1.05) and transition).


Important CSS properties applied:

  • flexible display: flex.
  • space between item gap.
  • position: relative and absolute to layer the SVG lines behind nodes.
  • Notice: pointer-events: none on the SVG to stop user interaction.

  • To have an excellent modern font, Google Fonts (Poppins) is imported.

    • When the SVG <line> elements have stroke-dasharray, they produce dashed lines between the nodes.


    Accessibility

    • Using semantic tags (<nav>, <footer>, <a>) for assistive technologies.
    • Ensuring that navigation links have descriptive text and emojis to enhance understanding.
    • Maintaining a clear visual hierarchy with consistent heading structure and readable fonts.
    • Ensuring good color contrast between nodes and background for readability.

    Content Page

    Technical Description


    1. HTML Structure:


  • The structure is enclosed within the <main> tag that has a semantic value of defining the main content of the page. This enhances the search engine optimization and compatibility with screen readers.

  • All personal topics (should be single such as Greenhouse Gas Emissions, Rising Temperatures) are aggregated with the use of <section> tags, which add a more documents structuring as well as modularity to the contents.

  • Titles will be formatted in terms of using <h1> and <h2> tags to maintain a correct hierarchy of the headings.

  • The description of all cause and effects is explained with <p> (paragraphs) or <ul> lists to enhance readability and semantics.


    • 2. CSSs usage:


  • Google Fonts (Poppins) were used to import fonts so that the whole thing would appear clean and professional.

  • It is already out-of-the-box responsive to mobile devices since the grid system places the layout, though only a few custom classes of the grid have been utilised in this case.

  • Margin top and padding classes are also used to give physical space between the sections, which facilitates UX and makes content visible at a glance.


    • 3. Navigation:


  • The navigation bar is made through the means of a <nav> tag with the core of the anchor tags that have the references to the other pages (Home, Volunteer, Team, etc.).

  • A special class custom-navbar2 may be stylish by oneself through the use of external stylesheet (style.css).


    • 4. Footer:


  • At the bottom of the page where we want to keep consistent branding and credit information we have a <footer> tag. This promises to make legal as well as contributor information visible throughout the site.
