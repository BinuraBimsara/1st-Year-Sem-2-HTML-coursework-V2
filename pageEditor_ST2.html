Home Page

Technical Description

  • <div class="text-container">

    Used to structure the hero section with a two-column layout: an image on the left and a headline + paragraph on the right. Flexbox ensures content is aligned side by side and stays responsive.

  • <div class="cards-container">

    A flex wrapper that holds four content cards. It uses flex-wrap and gap to manage spacing and allow responsive stacking on smaller screens.

  • <div class="card">

    Each card contains an image, a title, and a "Learn more" link. It’s styled with border-radius, background color (dark transparency), and a hover effect using transform: translateY(-8px) for smooth animation.

  • <nav class="custom-navbar">

    A consistent horizontal navigation bar with links to other major site pages. Flexbox is used here too for alignment.

  • Google Fonts & External CSS

    The page imports the 'Poppins' font for a modern and clean aesthetic. External stylesheets are linked for consistent branding.

  • Image Optimization

    All images inside cards use object-fit: cover to fill their space without distortion and maintain visual balance.

Accessibility

  • Each image uses alt attributes to describe content for screen readers.
  • Font contrast (white on dark) supports readability for low-vision users.
  • Navigation links are simple, clearly labeled, and use readable font sizes.

Table Page

Technical Description

  • <div class="pricing table">

    A custom-designed table using CSS Grid to display SDG 13 renewable energy actions by continent. Unlike normal <table> tags, this gives full control over layout and responsiveness.

  • grid template columns: repeat(5, 1fr)

    Creates a five-column layout with equal width, useful for aligning each column like a table (Region, Solar, Wind, etc.).

  • <div class="cell">

    Each grid cell contains data, styled with padding, transparency, and colored borders for visibility.

  • ✔ and ✘ Icons

    Used instead of text for accessibility and clean visual feedback. Styled with separate .tick and .cross classes for color coding.

  • <a href="...">

    Each row ends with a hyperlink styled as a button leading to external regional climate data pages. Opens in a new tab with target="_blank".

  • Responsive Container

    The full table is wrapped in a <div class="container"> with max width and padding to keep layout clean on all devices.

  • Bootstrap CDN

    Bootstrap 5 is linked via CDN but used minimallyprimarily kept for optional responsiveness and compatibility.

Accessibility

  • High-contrast design ensures readability (white on green/red backgrounds).
  • Symbols (✔, ✘) support quick visual scanning for users with cognitive differences.
  • Table layout adapts to mobile and desktop due to CSS Grid and container width controls.

Content Page: What is Climate Change?

Technical Description

  • <main>

    The main content area holds six well structured sections using semantic HTML. It explains climate change through a mix of headings, lists, and paragraphs.

  • <section>

    Each section is dedicated to a specific sub topic (definition, effects, science, etc.), improving readability and semantic structure.

  • <ul> and <li>

    Unordered lists are used to break down information into bite-sized points. This improves user comprehension and screen reader accessibility.

  • <h2> and <h4>

    Used to logically organize the page hierarchy. The large <h1> grabs attention at the top, while <h2> and <h4> help categorize the information clearly.

  • <link> (Google Fonts)

    Adds Poppins font to give a modern, clean look across all textual elements, improving the reading experience.

  • Custom CSS Classes

    Classes like .contentpages and .custom-navbar2 maintain layout consistency with the rest of the site. Colors and spacing match the SDG climate theme.

  • Bootstrap Integration

    Bootstrap 5 is included via CDN but is only lightly used (e.g. .container, .text-center, and .mb-5) to support mobile friendly and centered layout.

Accessibility

  • Uses semantic tags like <section> and <h2> for screen reader support.
  • Structured information in list format for better readability and focus.
  • Readable contrast between text and background using simple black on white sections.

Issues and Lessons


Making Sure of Responsive Design
Responsive design on the part of the site incorporating the multiple screen sizes was one of the crucial problems. This called on the judicious application of the CSS Flexbox and Grid systems. In the process, they got to discover the effective way in which responsive units and media queries can be used (Duckett, 2011).


Consistency in the Design
It was also hard to keep the fonts, buttons and card layouts similar on various pages, particularly when coding was done manually. The answer was to reuse CSS classes and design system with external styles (Marcotte, 2014).


Accessibility Implementation
It was essential to make the site compatible to everyone including screen readers. This was achieved through semantic HTML, alternate text on images and brush color contrast (W3C, 2024). This experience led to the lesson of inclusive practices in designing.


InternalBits of Navigation and Site Problems
Broken links happened in the earlier phases due to the improper file paths and navigation menus that were not unified. Such problems were addressed by verifying all of the relative paths and testing each page individually (Nixon, 2013).


The work of Art and Performance Aesthetic Balance
The use of quality images even increased page loading speeds. The lesson I took away was to optimize images by compressing them, and also using object-fit and the lazy-loading skills (Freeman, 2020).

Compliance


The web content conforms to academic and institutional requirements very strictly. It does not comprise offensive, illegal, or bias-based content. Rather, it is aimed at informing about the climate change, which all is socially. responsible values.


The pages do not collect user data, so they are ethical since it guarantees privacy. The code utilized is all clean. It is non-invasive, and it is developed with encrypted HTML, CSS and JavaScript and does not have trackers or third party exploitative plugins.


All external sources are reliable and refer to UN, education organizations, and open-source hosting.
